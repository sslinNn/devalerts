# devalerts

Отправляет необработанные исключения Python прямо в чат Telegram. Без бэкенда,
без аккаунта, без базы данных — только свой токен бота.

## Установка

    uv add devalerts

(или `pip install devalerts`, если не используете uv)

## Использование

1. Создайте бота через [@BotFather](https://t.me/BotFather) и получите токен.
2. Один раз напишите своему боту (или добавьте его в группу), чтобы он мог писать вам в ответ.
3. Узнайте свой chat id — напишите [@userinfobot](https://t.me/userinfobot), либо вызовите
   `https://api.telegram.org/bot<TOKEN>/getUpdates` после шага 2 и прочитайте `message.chat.id`.
4. В своём приложении, как можно раньше:

```python
import devalerts

devalerts.init(bot_token="123456:ABC-DEF...", chat_id=123456789)
```

Готово — теперь любое необработанное исключение (включая исключения в
потоках) также прилетает в ваш чат Telegram.

## Группировка, rate limiting и дашборд

Исключения группируются по fingerprint (тип исключения + файл + строка, где
оно возникло) в локальном SQLite-файле (`~/.devalerts/state.db`). Каждая
группа шлёт не больше одного сообщения в Telegram за `rate_limit_seconds`
(по умолчанию 300); повторы внутри этого окна считаются, но не отправляются,
а следующее сообщение по этой группе указывает, сколько раз она повторилась.
Старые группы (не встречавшиеся 7 дней) удаляются автоматически. Окно
настраивается через `init()`:

```python
devalerts.init(bot_token="...", chat_id=123456789, rate_limit_seconds=60)
```

Посмотреть, что сгруппировано и что сейчас подавлено rate-limit'ом:

```
uv run devalerts dashboard
```

## Ручная отправка пойманного исключения

```python
try:
    risky_call()
except Exception:
    devalerts.report()  # отправляет текущее обрабатываемое исключение
```

или:

```python
with devalerts.capture():
    risky_call()  # отправляет при исключении, затем пробрасывает его дальше
```

`capture` также работает как декоратор, так что тело функции трогать не нужно:

```python
@devalerts.capture()
def risky_call():
    ...
```

## FastAPI / Starlette / любое ASGI-приложение

Excepthook из `init()` не увидит ошибки запросов — фреймворк сам ловит их
внутри себя, чтобы вернуть ответ 500, так что до процесса ничего
"необработанного" не долетает. Вместо этого используйте ASGI-мидлварь:

```python
app.add_middleware(devalerts.ASGIMiddleware)
```

Отправляются только исключения, которые реально дошли до серверной ошибки —
роутинговые 404 и явно поднятые `HTTPException` фреймворк уже превращает в
ответ до того, как их увидит мидлварь.

## Чего это НЕ делает (осознанно)

- Группировка/rate limiting — только локально и внутри процесса (SQLite-файл,
  без сервера), дашборд — CLI-таблица, а не веб-интерфейс.
- Нет бэкенда, нет аккаунтов — каждый пользователь запускает своего бота.
- Только базовая редакция секретов (несколько частых паттернов токенов) — не
  полагайтесь на неё для чувствительных production-данных.
- Нет автоматического набора тестов — проверено вручную только при реализации.
